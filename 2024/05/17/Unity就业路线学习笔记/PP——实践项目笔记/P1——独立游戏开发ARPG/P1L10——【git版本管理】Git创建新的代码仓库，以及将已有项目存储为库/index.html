P1L10——【git版本管理】Git创建新的代码仓库，以及将已有项目存储为库

Github Desktop 创建新的代码仓库

1.认识界面

​​

2.创建新的存储仓库

点击从本地磁盘上创建新存储库

​​

在弹出窗口中填写关键信息

​​

我的填写如下，路径可以自己找一个磁盘新建一个文件夹用于测试

​​

点击Create repository 创建仓库后，创建成功，界面变为如下图所示

​​

我们可以点击窗口中的在资源管理器中查看存储库文件按钮

​​

便可以打开仓库所在位置

​​

此时，我们便可以在这个文件夹中创建Unity工程，或添加想要被版本控制的文件了

3.为新建仓库添加文件，上传版本节点

我们在次路径中创建一个测试用的文本文件，里面随便编辑一些内容

​​

回到Git窗口中

​​

​​

点击Commit to main 提交到主分支后，窗口刷新，表示当前版本没有修改

​​

切换到History版本历史页签

​​

4. 同步到远端GitHub仓库中

注意：做了以上操作，这些版本信息只是记录在了本地，并没有同步到远端Github仓库

同步到Github仓库

​​

点击后弹出窗口

​​

点击Publish repository 推送后 稍等一会，让其上传结束，界面将变为

​​

此时再在网页上查看自己的Github主页，就可以看到刚才推送的内容了

​​

在网页上点击这个项目，便可以看到该项目的详细内容

​​

5. 内容修改或内容添加

我们修改或添加内容，Git窗口中都会更新当前的变化情况，重复刚才的步骤提交节点，上传即可

​​

想要同步云端Github，点击Push origin 推送远端即可

​​

点击后，稍等一会让其上传成功，既可以在Github网页上看到最新内容

​​

Github Desktop 将已有项目存储为库

1.新建仓库

点击左上角File，在点击新建存储库

​​

在弹出窗口中选择对应的工程路径，之后选择Create repository创建存储库即可

（注意：仓库名一定是工程文件夹名，路径一定是工程文件夹的上一级）

​​

如果工程文件夹中存在大文件，会弹出该窗口

​​

此时我们切换到History页签，会看到初始节点中就包含了Unity工程中的所有内容

​​

2.上传到Github云端存储

此时我们可以直接选择推送存储库到云端即可

​​

弹出窗口无需做修改，直接推送即可

​​

稍等一会（根据你的工程大小，等待时间不等，即可推送成功）

我们便可以在Github主页中看到新的存储库信息

​​

点击它，便可以看到详细内容

​​

这样我们就把已有工程作为了Git的存储库项目，之后修改内容，便可以用之前学习的知识，重新创建节点，根据需求再次推送到云端即可了
